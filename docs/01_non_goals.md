# ClosedLoop：非目标与边界（Non-Goals）

## Purpose

这份文档回答：哪些东西现在明确不做，避免范围膨胀，方便做取舍与答辩。

## Non-Goals（不做什么）

- 不做复杂推荐算法优化
- 不做多 Agent 架构
- 不做重前端/复杂交互（以单页 + 少量按钮为主）
- 不做实时导航/路况

## Scope Boundary（边界约定）

- “可执行”优先于“最优推荐”：先保证闭环可跑，再谈效果优化
- “本地生活行程”以 4–6 小时为主，不扩展成全天/多日旅行规划
- 预订/排队/通知先以 Mock 为主，接口设计保留可替换真实平台能力
- 地图只做标点与路线展示，不做导航级别功能

## Not Now（以后再说）

- 个性化长期画像与跨 session 偏好学习
- 实时库存/实时排队时间的联动
- 多城市/跨城行程与交通优化
- 多语言与复杂可访问性改造

## Checklist

- 需求讨论或评审中，能用本页快速拒绝超范围项
- 任何新增需求必须明确落在“闭环能力”或“可验证指标”里
