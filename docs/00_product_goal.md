# ClosedLoop：执行型本地生活 Agent（目标稿）

## Purpose

这份文档回答：我们要把产品做成什么样（用户看到的体验、核心价值、Demo怎么演）。

## TL;DR

一句话目标：用户说一句话 → 系统生成“可执行行程” → 用户确认 → 自动完成预订/排队/通知（Mock） → 出错可自动修复（fallback/replan）。

## Intro（首屏自我介绍）

建议首屏文案：

我是你的本地探索：周末闲时活动规划助手。你只需要说一句话，比如：今天下午有空，想和老婆孩子/朋友出去玩几个小时，别离家太远，帮我安排一下。

## Intake（补问：让约束更完整）

- 是否有忌口/过敏（你/家人/朋友）
- 人数（party size）
- 若是亲子：孩子年龄（方便筛选更合适的活动与餐厅）

## Goals（要做到）

- 从自然语言需求生成 4–6 小时本地行程
- 支持组合消费（activity + food + extra）
- 提供一键执行（Mock 预订/排队/通知）
- 具备失败处理（fallback）与重规划（replan）
- 支持局部替换（replace）与自然语言修改（chat）

## Use Cases（核心场景）

### 家庭场景

- 输入：带老婆孩子（5岁）、轻食、下午 4–6h、别太远
- 期望：亲子乐园 → 轻食餐厅 → 甜品（可选）

### 朋友场景

- 输入：4人聚会（2男2女）
- 期望：展览/Citywalk → 餐厅 → 咖啡/小吃

## User Flow（用户流程）

```text
输入一句话
  ↓
生成方案（Bundle）
  ↓
用户微调（Replace / Chat）
  ↓
确认执行（Approval）
  ↓
执行（成功/失败）
  ├─ 成功 → 完成
  └─ 失败 → Fallback/重规划 → 完成
```

## UI/UX（产品形态）

### 单页结构（推荐 Streamlit / 简单 Web）

```text
[左] 地图（标点 + 路线）
[右] 行程卡片（可替换）
[底] Agent 输入框（自然语言修改）
```

### 行程卡片（核心）

```text
✨ 亲子轻松下午（5h）

14:00  🎠 室内乐园        [换]
16:30  🥗 轻食餐厅        [换]
18:00  🍰 甜品（可选）    [移除]

💡 可选加购：
🍰 甜品（+¥20） [加入]

[确认并安排]
```

### 执行过程（差异点）

```text
⏳ 正在安排…

✔ 乐园预约成功
❌ 餐厅满位
🔄 已替换为：餐厅B ✔
```

## Demo Script（必须按这个演）

1. 输入需求
2. Agent 补问：忌口/过敏、人数、孩子年龄
3. 展示方案（含候选餐厅与每家推荐套餐）
4. 点击“换餐厅”
5. Chat：“换便宜一点”
6. 点击执行 → 弹窗确认下单
7. 出现预约失败
8. 自动替换/或用户指定换一家
9. 完成 → 一键分享给朋友/家人

## Pitch（一句话，写在答辩）

我们不是推荐系统，而是一个能在不确定环境中完成任务的执行型智能体系统。

## Metrics（先定口径，后再做埋点）

- Plan Success Rate（生成方案并通过校验的比例）
- Execution Success Rate（执行步骤全部成功的比例）
- Fallback Rate（进入 fallback 的比例）
- Latency（端到端延迟：从输入到可执行方案/执行完成）

## Checklist

- 能从一句话生成 4–6h 行程（含 activity + food + optional extra）
- UI 可进行 Replace/Chat 微调
- 有确认与执行态展示
- 执行失败能自动修复并给出替换结果
